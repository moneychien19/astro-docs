這個教學將會使用 GitHub 儲存我們的儲存庫，並連接到網路主機，你也可以使用其他任何你想要的線上 git 服務。

:::note
如果你已經熟悉 git 並擁有習慣的工作流程，你可以用你想要的方式為專案建立一個新的 GitHub 儲存庫，並跳到下一頁：["將網站部署到網路上"](/zh-tw/tutorial/1-setup/5/)。
:::

## 在 GitHub 上建立儲存庫

將程式碼儲存到 GitHub 上的方式有很多種，這個教學將會教你使用不必用到命令列的方法。

1. 在瀏覽器上登入 GitHub 並點擊右上角的 <kbd>+</kbd> 建立新的儲存庫。

2. 為你的儲存庫選擇一個名字，這個名字可以不用和專案資料夾的名字相同。

3. 這時會出現很多選項，但是你不用更改任何預設的選項。將頁面往下滑並按 <kbd>Create Repository</kbd> 以建立儲存庫。

4. 你會看到更多後續的設定選項，但你不需要使用它們，你只需要記下這個儲存庫的網址，就可以離開這個頁面。


## 將本地的程式碼提交到 GitHub 上

在上一個章節中，你編輯了頁面的內容，這代表你專案中的檔案已經被更改了，這時應該會有一個數字出現在 VS Code 的 "原始檔控制" 圖示上。這個分頁也是你在更新檔案進度到 GitHub 上時會常常用到的分頁。


1. 點擊 VS Code 中的原始檔控制分頁以查看已更改檔案的列表，如果你看到提示安裝 `git` 的訊息，依照指示安裝並重新載入 VS Code。

2. 點擊提交訊息上方的 <kbd>•••</kbd> 選單並選擇 <kbd>遠端</kbd> > <kbd>新增遠端存放庫</kbd>。

3. 選擇 <kbd>從 GitHub 新增遠端存放庫</kbd>。如有必要，請跟隨身份驗證的指示，然後回到 VS Code 重複此步驟。 

4. 你應該會看到你在 GitHub 上的所有儲存庫列表，選擇你為這個專案建立的儲存庫。如果你沒有看到你的專案，你可以直接貼上 GitHub 儲存庫的網址。你還會被要求給予本地端專案一個名字，這時你可以選擇任何你想要的名字。

5. 在選單的最上方會有一個地方可以輸入**提交訊息**（關於此次檔案改動的描述），輸入 `initial commit` 並按下`提交`按鈕。

7. 你會看到一個提示你沒有 "staged" 提交的訊息，並詢問你是否要暫存它們，按下 <kbd>永遠</kbd> 並繼續。 

8. 最後，這些更改過的檔案列表會被 <kbd>同步變更</kbd> 按鈕取代，點擊它以將提交送到 GitHub 上。


### 在 GitHub 中查看你的專案

你可以到 GitHub 查看個人檔案下的儲存庫列表，以驗證你的專案被成功儲存到 GitHub。選擇你新建立的儲存庫並驗證它裡面包含了你 Astro 專案裡的檔案。
